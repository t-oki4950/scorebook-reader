# Codexへのスコアブック仮読み取り依頼

以下の画像は、原則としてスコアブックの「1打席分のマス」です。
画像を読解し、`outputs/provisional_reading.json` に保存できる仮読み取りJSONを作成してください。

画像ファイル:

```text
/Users/okitoyokazu/Documents/Codex/2026-06-10/ai-ai-web-json-csv-markdown/scorebook-reader/inputs/images/20260613-102204-pasted-scorebook.png
```

## 目的

この作業の目的は、チーム成績DBを完成させることではありません。
1打席分の画像を仮読み取りし、人間がStreamlit画面で添削できる形のJSONを作ることです。
人間の添削結果は、次回以降の読解ルール・事例として蓄積します。

## 出力先

`outputs/provisional_reading.json`

## 出力形式

```json
{
  "schema_version": "1.0",
  "source_image": "inputs/images/画像ファイル名",
  "status": "provisional",
  "at_bats": [
    {
      "id": "inning1-batter1",
      "inning": 1,
      "batter_number": 1,
      "symbols": ["読み取った記号、打撃記録、走塁記録"],
      "interpretations": ["記号の解釈"],
      "pitch_sequence": ["ー", "ー", "✗"],
      "count_summary": "2ボール1ストライク",
      "cumulative_pitch_count": 0,
      "batting_result": "打撃結果",
      "out_count": "",
      "runner_events": [],
      "notes": "曖昧な点や根拠"
    }
  ]
}
```

## 読解時の注意

- 断定できない箇所は `notes` に推測として明示してください。
- まず「投球の記号と順序」「カウント」「打撃結果」「走塁状況」を考えてください。
- 小さい数字は相手投手の累計投球数として扱ってください。
- ボールカウント欄が小さくても、見える投球記号は必ず `symbols` に入れてください。
- 投球記号の順序が読める場合は、`pitch_sequence` に順番通り入れてください。
- カウントが推定できる場合は、`count_summary` に「2ボール1ストライク」のように書いてください。
- `ー`, `✗`, `二重✗`, `△`, `B`, `K`, `SK` などを見つけたら、打撃・走塁記録と同じ打席データ内の `symbols` にも入れてください。
- 判読が不確実な投球記号は、`?` または `らしき記載` を付けて `symbols` に残し、`notes` に不確実性を書いてください。
- 「’」は独立した投球ではなく、同時イベントの印として扱ってください。
- `B` や `SK` は最後の投球を兼ねるため、投球数を二重計上しないでください。
- 1打席画像の中で読めない領域がある場合も、見える範囲だけでJSONを作ってください。

# スコアブック読解ルールコンテキスト

以下は、草野球・軟式野球スコアブック画像を仮読み取りするための蓄積ルールです。
AIはこの内容を優先し、曖昧な箇所は推測として明示してください。

## 単独記号ルール

- `1`: 投手（守備番号）
  - 補足: 守備番号1は投手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `2`: 捕手（守備番号）
  - 補足: 守備番号2は捕手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `3`: 一塁手（守備番号）
  - 補足: 守備番号3は一塁手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `4`: 二塁手（守備番号）
  - 補足: 守備番号4は二塁手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `5`: 三塁手（守備番号）
  - 補足: 守備番号5は三塁手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `6`: 遊撃手（守備番号）
  - 補足: 守備番号6は遊撃手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `7`: 左翼手（守備番号）
  - 補足: 守備番号7は左翼手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `8`: 中堅手（守備番号）
  - 補足: 守備番号8は中堅手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `9`: 右翼手（守備番号）
  - 補足: 守備番号9は右翼手を表す。数字単独の場合は守備位置への打球方向または打撃結果の一部として読む。
- `1-3`: ピッチャーゴロ、ファースト送球アウト（守備・打球記録）
  - 補足: 投手が処理し、一塁手へ送球して打者走者をアウトにした記録。
- `6-4-3`: ショートゴロ、ダブルプレー（守備・打球記録）
  - 補足: 遊撃手、二塁手、一塁手の順に送球して成立した併殺。
- `4E`: セカンドのエラーによる出塁（守備・打球記録） / 別表記: 4 E
  - 補足: 二塁手の失策により打者走者が出塁した記録。
- `3 T.O`: 一塁手によるタッチアウト（守備・打球記録） / 別表記: 3T.O, 3 TO
  - 補足: 一塁手が打者走者または走者にタッチしてアウトにした記録。
- `ー`: ボール（投球記号） / 別表記: -
  - 補足: 投球がボールだったことを示す。
- `✗`: 見逃しストライク（投球記号） / 別表記: ×, x
  - 補足: 打者が見逃したストライクを示す。
- `二重✗`: 空振りストライク（投球記号） / 別表記: ✗✗, ××, xx
  - 補足: 空振りによるストライクを示す。
- `△`: ファール（投球記号）
  - 補足: ファールを示す。
- `B`: フォアボール（投球記号）
  - 補足: 打席の最終記号がBの場合はフォアボールとして読む。
- `SK`: 空振り三振（投球記号）
  - 補足: 打席の最終記号がSKの場合は空振り三振として読む。
- `◇`: バント（その他の記号）
  - 補足: バントを示す。走者状況やアウト記録と組み合わせてスクイズなどを判定する。
- `WP`: ワイルドピッチ（その他の記号）
  - 補足: 暴投を示す。「’」と対応している場合、その投球と同時に発生したイベントとして読む。
- `Ⅰ`: 1アウト目（その他の記号） / 別表記: I
  - 補足: その回の1つ目のアウトを示す。
- `Ⅱ`: 2アウト目（その他の記号） / 別表記: II
  - 補足: その回の2つ目のアウトを示す。
- `Ⅲ`: 3アウト目（その他の記号） / 別表記: III
  - 補足: その回の3つ目のアウトを示す。
- `小さい数字`: 相手投手の累計投球数（その他の記号）
  - 補足: マス内に小さく書かれた数字は相手投手の累計投球数として読む。
- `中央のl`: 残塁（その他の記号） / 別表記: 中央のL, 中央l, l
  - 補足: マス目中央のlはこの回の残塁を示す。
- `右下の斜め二重線`: 3アウトで攻撃終了（その他の記号） / 別表記: 右下斜め二重線, 斜め二重線
  - 補足: 右下に書かれた斜め二重線は3アウトで攻撃が終了したことを示す。
- `マス目中央の●または塗りつぶし`: 得点、ホームイン（その他の記号） / 別表記: ●, 塗りつぶし
  - 補足: マス目中央の●または塗りつぶしは得点、ホームインを示す。
- `’`: 同時イベント（その他の記号） / 別表記: '
  - 補足: 独立した投球ではなく、その投球に付随する同時イベントを示す。
- `IP`: インプレー（投球記号） / 別表記: インプレー
  - 補足: 打者がフェアグラウンドへ打球を飛ばし、打撃結果の判定に進んだ投球を示す。
- `1H`: 投手への内野安打（守備・打球記録） / 別表記: 1 H, 投手内野安打
  - 補足: 投手方向への打球で、打者走者が安打として一塁へ出塁した記録。
- `DS`: DS（走塁）
  - 補足: ダブルスチール

## 位置ルール

- 正方形の右下: 1塁到達
  - 補足: 1打席マスを野球場に見立てたときの1塁側。打者走者または走者が1塁に到達したことを示す位置。打者の出塁結果もこの付近に書かれやすい。
- 正方形の右上: 2塁到達
  - 補足: 1打席マスを野球場に見立てたときの2塁側。走者が2塁に到達したことを示す位置。
- 正方形の左上: 3塁到達
  - 補足: 1打席マスを野球場に見立てたときの3塁側。走者が3塁に到達したことを示す位置。
- マス目中央の菱形: 走者の進塁状況
  - 補足: 中央の菱形は、ランナーがベースを回る様子を表す。進塁線や赤線は、右下の1塁、右上の2塁、左上の3塁へ向かう走者移動として読む。
- マス目中央: 本塁生還、得点、残塁の判定位置
  - 補足: 中央の●または塗りつぶしは本塁生還・得点として読む。中央のlは残塁として読む。
- 正方形の右下または中央の菱形内: 打者の打席結果
  - 補足: 打者がどう出塁したか、またはどの方向へ打ったかは、右下の1塁側または中央の菱形内に書かれやすい。例: 右下の⑧と1塁までの線はセンター前ヒットの可能性が高い。
- マス内の小さい数字: 相手投手の累計投球数
  - 補足: 投球記録とは分離し、累計投球数として読む。

## 複合ルール

### スクイズ成功、打者タッチアウト
- 優先度: 95
- 条件:
  - ◇がある
  - 三塁走者がホームインしている
  - 打者の結果にT.Oがある
- 解釈: スクイズ成功。三塁走者は得点し、打者はタッチアウト。

### 3アウトで攻撃終了
- 優先度: 90
- 条件:
  - Ⅲがある
  - 右下斜め二重線がある
- 解釈: 3アウトで攻撃終了。

### WPとアポストロフィの対応
- 優先度: 90
- 条件:
  - WPがある
  - WPが「’」と対応している
- 解釈: その投球と同時にワイルドピッチが発生したと解釈する。

### 同時イベントの投球数扱い
- 優先度: 88
- 条件:
  - 「’」がある
  - 「’」がWPなどのイベントと対応している
  - 同じ投球欄に投球記号がある
- 解釈: 「’」で示された同時イベントは独立投球として数えず、対応する投球1球の付随イベントとして扱う。

### BやSKは最後の投球を兼ねる
- 優先度: 88
- 条件:
  - 打席の最終記号がBまたはSKである
  - 同じ打席の投球数を確認している
- 解釈: BやSKは打席結果であると同時に最後の投球を表すため、別の投球として二重計上しない。

### レフトへの三塁打
- 優先度: 85
- 条件:
  - 守備番号7がある
  - 進塁線が三塁まで到達している
- 解釈: レフトへの三塁打。

### 最終記号Bはフォアボール
- 優先度: 80
- 条件:
  - 打席内の最終的な打撃結果記号がBである
- 解釈: 打撃結果はフォアボール。

### 最終記号SKは空振り三振
- 優先度: 80
- 条件:
  - 打席内の最終的な打撃結果記号がSKである
- 解釈: 打撃結果は空振り三振。

### アポストロフィは同時イベント
- 優先度: 70
- 条件:
  - 「’」がある
- 解釈: 「’」は独立した投球ではなく、その投球に付随する同時イベントとして扱う。
